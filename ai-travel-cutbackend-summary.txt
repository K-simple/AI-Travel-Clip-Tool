3. Backend routers
------------------
- backend/routers/template.py
  - Uploads template videos
  - Uses services.scene_detector.detect_scenes to split template into slots
  - Saves template file and slot metadata to Template table
- backend/routers/assets.py
  - Uploads material video assets
  - Uses services.scene_detector.get_video_duration and extract_frame for thumbnail
  - Uses services.asset_analyzer.analyze_asset to segment asset and analyze labels
  - Saves Asset and segment info to Asset table
- backend/routers/match.py
  - Reads template slots and all asset segments
  - Uses services.matcher.match_slots to assign best matching segment for each slot
  - Saves a Project with matched timeline
- backend/routers/subtitle.py
  - Receives uploaded file for subtitle recognition
  - Saves temp file and calls services.subtitle_gen.transcribe
  - Generates SRT via services.subtitle_gen.generate_srt
- backend/routers/export.py
  - Exports final video from project timeline
  - Finds template video audio source
  - Calls services.video_exporter.export_video

4. Backend service details
--------------------------
- backend/services/scene_detector.py
  - Uses ffprobe + FFmpeg to get duration and extract thumbnail frame
  - Uses scenedetect ContentDetector to split videos into scenes
  - Returns slot objects with start/end/duration/thumbnail/tags/shot_type
- backend/services/asset_analyzer.py
  - Loads CLIP model and preprocess for image analysis
  - Analyzes scene type, shot type, and presence of a person from thumbnail image
  - Evaluates clip quality with OpenCV Laplacian variance
  - Adds scene_tags, shot_type, has_person, quality_score to asset segments
- backend/services/matcher.py
  - Match algorithm computes scores from duration, tags, shot type, quality
  - Selects one best segment for each template slot
  - Avoids duplicate segment reuse
- backend/services/subtitle_gen.py
  - Loads WhisperModel("base", device="cpu", compute_type="int8")
  - transcribe(audio_path) returns timestamped segments
  - generate_srt writes SRT file with hh:mm:ss,ms timing
- backend/services/video_exporter.py
  - export_video builds final video from matched timeline
  - 1) cuts each segment to resolution and pad background
  - 2) concatenates video clips with ffmpeg concat
  - 3) extracts template audio and merges it into merged video
  - 4) optional subtitle recognition and burn-in via burn_subtitles
  - burn_subtitles uses FFmpeg subtitles filter and a temporary SRT file
  - if add_subtitles is false, or subtitle fails, output copies with_audio

6. Key problems to fix next
---------------------------
- Audio handling in export_video
  - export_video currently drops material clip audio with `-an` when cutting clips
  - It then only uses template video audio for final export
  - As a result, original asset voice/music is lost and subtitles may be aligned to wrong audio
- Subtitle recognition source selection
  - export_video chooses `audio_source = template_video_path if template_video_path else with_audio`
  - If template video audio is background music or contains no speech, subtitle recognition is wrong
  - It should use actual speech audio from the final video or from matched asset clips, not template audio alone
- Subtitle burn-in reliability
  - burn_subtitles path escaping may fail on Windows
  - Hard-coded font and style could be made configurable
  - If FFmpeg subtitle burn fails, output silently falls back to copying original video
- Missing frontend integration for API calls
  - Frontend components are placeholder UI and do not call backend endpoints
  - There is no confirmed flow for uploading templates/assets, running match, and exporting via UI

7. Recommendation for next AI

Fix the backend export and subtitle flow first. The next AI should:

  4. Add logging / error reporting when FFmpeg fails during export, audio extraction, or subtitle burn
